Why this site costs £0 to host — and why that’s the point

March 2026·5 min read

The Experiment is about testing whether online income is real without spending money to make money. So the site itself had to follow the same rule. Here’s how it’s built, what it costs, and why GitHub Pages is worth knowing about.

The rule the site has to follow

The Experiment tests nine income streams from zero. The whole premise is that online income should be accessible to anyone — not just people willing to spend £50/month on tools and hosting before they’ve earned a penny. If the site itself had a monthly hosting bill, that would quietly undermine the point.

So when it came to building the site, the brief was simple: £0 per month in infrastructure costs, indefinitely. No hosting package. No CMS subscription. No monthly anything.

“The site demonstrating how to earn income online shouldn’t itself require a subscription to exist.”

What GitHub Pages actually is

GitHub is a platform developers use to store and manage code. Every project (called a “repository”) can optionally be published as a live website through a feature called GitHub Pages. It’s completely free. Has been for years. There are no adverts, no bandwidth limits for typical content sites, and no hidden fees.

You build your site as plain HTML, CSS, and JavaScript files, push them to a GitHub repository, enable Pages in the settings, and the site is live within minutes at a github.io address. You can also connect a custom domain if you have one.

What GitHub Pages is good for: Content sites, documentation, portfolios, blogs, and any site that doesn’t need a database or server-side processing. Static HTML files load fast, are easy to update, and cost nothing to host.

What GitHub Pages isn’t good for: Anything requiring a backend — user logins, dynamic content, database queries, e-commerce checkouts. If you need a proper web application, you’ll need a different solution.

For comparison, a typical WordPress site on a shared hosting plan costs between £5 and £20 per month. A Squarespace or Wix subscription runs £10–£25 per month. Over a year that’s £60–£300 in infrastructure before the site has earned anything. For an experiment about whether online income is achievable from zero, that felt like the wrong foundation.

The honest tradeoffs

Free hosting isn’t without constraints. Being transparent about them:

No dynamic content. Every page on this site is a static HTML file. There’s no database, no login system, no server-side anything. This is fine for a content and documentation site — but it means things like a proper members area or dynamic income charts updating in real time aren’t possible without third-party tools.

The URL. The site currently lives at a github.io address rather than a custom domain. A custom domain costs around £10–£15 per year and can be connected to GitHub Pages easily. This is on the roadmap as the Experiment generates income — the first thing the site earns will go towards its own domain.

No CMS. Every update to the site requires editing HTML files directly and pushing to GitHub. There’s no visual editor, no drag-and-drop, no admin dashboard. This is a real constraint for non-technical users. Content is managed by Digital Revenue Studio for this reason.

✓ The verdict

GitHub Pages is the right choice for this experiment at this stage. Zero ongoing cost means zero overhead while the income experiments are finding their feet. When the site earns enough to justify a domain and potentially a hosting upgrade, that decision will be documented here too — just like everything else.

What changes when the Experiment earns more

The plan is straightforward. As the Experiment generates confirmed income, infrastructure gets upgraded in order of priority:

First: Custom domain (~£12/year) — still on GitHub Pages, just with a proper URL.
Second: Own Sender account and email address, separate from DRS.
Third: Evaluate whether a more capable hosting solution (with a backend) opens up income experiments that aren’t currently possible.

All of this will be documented in the income reports as it happens. The principle stays the same: nothing gets spent until the income justifies it.
